C++ Game Development: Множество способов убить орка (или героя)

Доклад Патриса Руа на CppCon 2024 о различных подходах к реализации боевой системы в играх на C++

Суть доклада

В этом докладе Патрис Руа рассматривает множество различных способов реализации, казалось бы, простой задачи в игровой разработке: расчет урона между атакующим и защитником (например, между орком и героем).

Докладчик показывает, как один и тот же игровой механизм можно реализовать с помощью:

  • Обычных функций и структур
  • Объектно-ориентированного подхода (наследование, виртуальные функции)
  • Композиции и компонентного подхода
  • Систем на основе данных (Data-Oriented Design)
  • Использования современных возможностей C++ (std::variant, std::visit)

Основная мысль доклада: не существует единственно правильного способа — каждый подход имеет свои преимущества и недостатки в зависимости от контекста, требований к производительности и поддерживаемости кода.

Примеры кода из доклада

Патрис показывает несколько подходов к расчету урона. Вот основные из них:

Подход 1: Простые функции

// Самый простой способ — обычная функция
                                int computeDamage(int attackerStrength, int defenderArmor) {
                                    int rawDamage = attackerStrength * 10;
                                    int mitigatedDamage = rawDamage - defenderArmor;
                                    return std::max(0, mitigatedDamage);
                                }

Подход 2: Объектно-ориентированный (полиморфизм)

class Entity {
                                public:
                                    virtual int getAttackPower() const = 0;
                                    virtual int getDefense() const = 0;
                                    virtual ~Entity() = default;
                                };

                                class Orc : public Entity {
                                public:
                                    int getAttackPower() const override { return 15; }
                                    int getDefense() const override { return 5; }
                                };

                                int combat(Entity& attacker, Entity& defender) {
                                    int damage = attacker.getAttackPower() - defender.getDefense();
                                    return std::max(0, damage);
                                }

Подход 3: Компонентный подход (Data-Oriented)

// Компоненты — простые структуры данных
                                struct HealthComponent { int hp; };
                                struct StrengthComponent { int strength; };
                                struct ArmorComponent { int armor; };

                                // Система, которая обрабатывает все сущности
                                void damageSystem(std::vector<Entity>& entities) {
                                    for (auto& e : entities) {
                                        if (e.has<StrengthComponent>() && e.has<ArmorComponent>()) {
                                            int damage = e.get<StrengthComponent>().strength 
                                                    - e.get<ArmorComponent>().armor;
                                            if (damage > 0) {
                                                e.get<HealthComponent>().hp -= damage;
                                            }
                                        }
                                    }
                                }

Подход 4: Использование std::variant (современный C++)

using GameEntity = std::variant<Orc, Hero, Goblin, Dragon>;

                                struct DamageVisitor {
                                    int operator()(const Orc& o) const { return o.strength * 2; }
                                    int operator()(const Hero& h) const { return h.attackPower; }
                                    int operator()(const Goblin& g) const { return g.damage; }
                                    int operator()(const Dragon& d) const { return d.fireBreath(); }
                                };

                                int getDamage(const GameEntity& entity) {
                                    return std::visit(DamageVisitor{}, entity);
                                }

Выводы из доклада

Когда что использовать?
  • Функции — для простых, изолированных расчетов
  • ООП — когда иерархия сущностей стабильна
  • Компоненты (ECS) — для больших игр с множеством типов
  • std::variant — для небольшого набора типов с разным поведением
Главные уроки
  • Нет "серебряной пули" — выбирайте под задачу
  • Производительность vs поддерживаемость — всегда компромисс
  • Современный C++ дает больше гибких инструментов
  • Профилируйте код перед оптимизациями
